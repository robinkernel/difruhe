# Start Here

Difruhe is not a tool.
Difruhe is not a platform.

Difruhe is a structural language for distinguishing what is merely proposed from what is valid, referenceable, and allowed to become real.

It exists for a specific boundary:

**the boundary between AI intent and real-world action.**

As AI systems move from generating language to triggering actions, the critical question is no longer only what a model can produce.

The critical question becomes:

- what is still only a candidate
- what has become a valid state
- what is admissible before execution
- what is allowed to become real

## Core claims

- Candidate is not action.
- State must come before intent.
- Admission must come before execution.
- Container validity matters.
- Execution success does not equal legitimacy.

## What Difruhe is trying to make explicit

Difruhe makes explicit a missing governance layer between proposed output and legitimate action.

It is designed to help distinguish:

- generation from admission
- candidate output from valid state
- execution from legitimacy

If that boundary is not explicit, stable, and verifiable, action systems will keep confusing success with validity.
