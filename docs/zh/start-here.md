# 从这里开始

Difruhe 不是一个工具。  
Difruhe 不是一个平台。

Difruhe 是一种结构语言，用来区分：

什么还只是被提出的候选内容，  
什么已经成为有效、可引用、并被允许进入现实的对象。

它要处理的是一个非常具体的边界：

**AI 意图与现实动作之间的边界。**

当 AI 系统从生成语言走向触发现实动作时，真正关键的问题，不再只是模型“能产出什么”。

真正关键的问题变成：

- 什么仍然只是 candidate
- 什么已经构成 valid state
- 什么在执行前是 admissible 的
- 什么被允许 become real

## 核心判断

- Candidate is not action.
- State must come before intent.
- Admission must come before execution.
- Container validity matters.
- Execution success does not equal legitimacy.

## Difruhe 想先讲清什么

Difruhe 想先讲清一层长期被忽略的治理层：

它位于 proposed output 和 legitimate action 之间。

它帮助我们区分：

- generation 与 admission
- candidate output 与 valid state
- execution 与 legitimacy

如果这个边界不被明确化、稳定化、可验证化，  
动作系统就会不断把“执行成功”误当成“状态有效”。
