# Capability 不是 Admissibility

## 为什么下一个 AI 瓶颈，不只是系统能做什么，而是它是否已经获得了让某件事成为现实的资格

过去几年，大多数 AI 讨论都围绕 capability 展开。

模型能不能更好地推理？  
agent 能不能规划得更长？  
系统能不能使用更多工具？  
它能不能拥有更强的 autonomy？  

这些问题当然成立。  
但它们已经不是最深的那一层了。

更深的问题是：

**什么东西，已经获得了成为现实的资格？**

这不是 capability 问题。  
这是 admissibility 问题。

一个系统也许能够生成一个计划、一个建议、一个 workflow、一个代码补丁、一个任务片段、一个机器人动作，或者一个科学假设。  
但这并不意味着系统已经获得了把它真正施加出去的资格。

而这正是这个领域仍然严重欠规格化的地方。

**Capability 问的是：系统能做什么。  
Admissibility 问的是：系统在一个有效状态下、在足够 authority 支撑下、在一个可信容器里，此时此刻可以做什么。**

一旦 AI 从语言生成走向对外部现实产生影响，这个差别就会变得决定性。

因为那时真正的失败模式，已经不只是：

“答案错了。”

更深的失败模式是：

**系统在没有资格的情况下改动了世界。**

这就是为什么 capability 本身不够。

一个成熟系统，至少必须把下面四件事分开：

- existence
- value
- admissibility
- execution rights

而大多数系统仍然把它们压成一团。

如果某个东西存在，它们就把它当成 live。  
如果某个东西有价值，它们就把它当成 urgent。  
如果某个东西 urgent，它们就给它 execution priority。  
如果某个东西被生成出来，它们就把它当成 ready for action。

系统就是这样开始漂移的。

更强的系统会做别的事。

它会说：

- 这也许是 plausible 的
- 这也许是 valuable 的
- 这也许值得被保留
- 但它仍然不是 current mainline
- 它仍然没有 admitted
- 它仍然不被允许成为现实

这才是真正治理的开始。

这里最关键的结构动作其实很简单：

**admission 必须先于 execution。**

不是只先于 planning。  
不是只先于 scoring。  
不是只先于 confidence。  
而是 admission。

这意味着系统必须问：

- 当前 state 是否有效？
- 行动 authority 是否真的已经成立？
- evidence 和 source 条件是否足够？
- 当前 container 是否仍然足够可信，能够承载 action？
- 这个 candidate 是否已经获得了被提升进现实的资格？

如果没有这个过渡，execution 就会变成一种带有力量的猜测。

而带有力量的猜测，不是可信 autonomy。  
那是危险的加速。

这也是为什么，future direction 和 current action 必须被不同地治理。

一个 future direction 可能在战略上是正确的，  
但它仍然可能对 present work 来说是 inadmissible。

它可能值得被正式保留。  
可能值得被保护，不被遗忘。  
可能值得继续在系统里保持可见。  
但这仍然不代表它值得获得当前 execution rights。

这就是下面两者的区别：

**保留未来**  
和  
**让未来冒充现在**

一个成熟系统，必须知道如何做到前者，  
同时不滑进后者。

下一代真正严肃的 AI stack，会在这里和今天的系统拉开差距。

它不会只优化 generation。  
它会治理 transition。

它不会只优化 proposal。  
它会治理 admission。

它不会只优化 action。  
它会治理“成为现实的资格”。

这就是为什么，下一层关键层，不是又一层 model layer。  
不是又一个 workflow wrapper。  
也不是又一个 agent abstraction。

它是一个 **admissibility layer**。

这一层决定：

- 什么仍然只是 candidate
- 什么变成 valid state
- 什么在 execution 前是 admissible 的
- 什么必须继续冻结
- 什么必须被拒绝
- 什么终于获得了进入世界的资格

AI 的未来，不会只由系统“能做多少”决定。

它会由系统是否能够分清：

**什么可以被生产出来**  
和  
**什么被允许成为现实**

来决定。

这就是 capability 和 admissibility 的区别。

而这个区别，就是下一场真正架构之战发生的地方。

**AI 不只是需要更强的 capabilities。  
它还需要一种方式，来决定哪些 capabilities 已经获得了成为现实的资格。**
