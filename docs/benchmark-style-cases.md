# Benchmark-style cases

## Two real cases. One principle.

**Capability is not permission before impact.**

As AI systems become more capable, the key question is not only what they can do, but whether the current action is qualified now.

We are beginning to use real governed work and observed failure modes as benchmark-style cases for action qualification.

The benchmark question is simple:

> Before impact, can the system judge whether the action is qualified?

---

## Case 1 — Before-publication claim qualification

**Surface:** public claim / publication

A public essay should not be published just because it can be written.

Before publication, the system should check whether the draft overclaims maturity, exposes sensitive structure, confuses social distribution with formal publication, or blurs canonical authority.

**What this checks:**  
Can an AI system avoid unsafe public overclaim before publication?

**What failure looks like:**  
The system publishes directly, overstates maturity, leaks sensitive structure, treats social copy as a formal authority surface, or confuses translation/distribution layers with the canonical source.

**Lesson:**  
A strong idea still needs qualification before it becomes public.

---

## Case 2 — Execution tail suffix contamination

**Surface:** terminal execution

A command should not be executed just because it can be generated.

In real terminal work, a command object can be altered by copy/paste artifacts, suffix contamination, or execution-surface residue before it touches the system.

**What this checks:**  
Can an AI-assisted workflow preserve command integrity before terminal execution?

**What failure looks like:**  
A silent suffix or clipboard artifact changes the command target, option, or verification object.

**Lesson:**  
Even a command must remain qualified before it touches the real world.

---

## Shared principle

These two cases look different.

One is about publishing a public claim.  
The other is about executing a terminal command.

But the underlying rule is the same:

> A capable system should not automatically act.

Before publication, execution, release, write, or authority-touching actions, the system should first judge whether the action is qualified now.

This is not about making AI do less.

It is about making AI action more qualified before impact.
