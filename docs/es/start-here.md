# Empieza aquí

Difruhe no es una herramienta.  
Difruhe no es una plataforma.

Difruhe es un lenguaje estructural para distinguir:

qué sigue siendo solo una propuesta,  
qué ya es válido, referenciable,  
y qué está permitido para volverse real.

Existe para una frontera muy específica:

**la frontera entre la intención de la IA y la acción en el mundo real.**

A medida que los sistemas de IA pasan de generar lenguaje a activar acciones reales, la pregunta crítica ya no es solo qué puede producir un modelo.

La pregunta crítica pasa a ser:

- qué sigue siendo solo un candidate
- qué ya constituye un valid state
- qué es admissible antes de la ejecución
- qué está permitido become real

## Core claims

- Candidate is not action.
- State must come before intent.
- Admission must come before execution.
- Container validity matters.
- Execution success does not equal legitimacy.

## Qué quiere hacer explícito Difruhe

Difruhe busca hacer explícita una capa de gobernanza que suele quedar oculta:

la capa entre proposed output y legitimate action.

Ayuda a distinguir:

- generation y admission
- candidate output y valid state
- execution y legitimacy

Si esa frontera no es explícita, estable y verificable, los sistemas de acción seguirán confundiendo éxito de ejecución con validez.
